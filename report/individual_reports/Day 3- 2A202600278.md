```python
class Tool:
    def __init__(self, name, func, description):
        self.name = name
        self.func = func
        self.description = description

    def run(self, *args, **kwargs):
        return self.func(*args, **kwargs)
```

**2. ReAct Loop (simplified in main.py)**

```python
while True:
    thought = llm.generate_thought(context)
    action = llm.generate_action(thought)

    if action["type"] == "tool":
        observation = tools[action["name"]].run(**action["args"])
        context.append(observation)
    else:
        break
```

---

### Documentation

Hệ thống hoạt động theo mô hình **ReAct (Reasoning + Acting)**:

1. **Thought**: LLM phân tích yêu cầu người dùng (ví dụ: tìm phương tiện rẻ nhất).
2. **Action**: Chọn tool phù hợp (`search_bus`, `search_flight`, `search_train`).
3. **Observation**: Tool trả về dữ liệu (giá, thời gian, hãng).
4. **Loop**: LLM tiếp tục reasoning dựa trên observation.
5. **Final Answer**: Tổng hợp kết quả tốt nhất.

Luồng này giúp agent không chỉ “trả lời” mà còn **thực sự thực thi hành động**.

### Problem Description

Agent bị lặp vô hạn với hành động:

```

```

→ Không có tham số hợp lệ nhưng vẫn tiếp tục gọi tool.

### Diagnosis

Nguyên nhân chính:

* Prompt chưa ràng buộc chặt format `Action Input`
* Tool spec không validate input
* LLM không hiểu rõ required parameters

=> Đây là lỗi **prompt design + tool interface**, không phải do model.

## III. Personal Insights: Chatbot vs ReAct (10 Points)

### 1. Reasoning

ReAct tốt hơn chatbot ở:

* Chia nhỏ bài toán (search → filter → compare)
* Có khả năng “suy nghĩ từng bước” (Thought)
* Tránh hallucination vì dựa vào tool thật

Ví dụ:

* Chatbot: đoán giá vé
* ReAct: gọi API → trả dữ liệu thật

---

### 2. Reliability

ReAct có thể **kém hơn chatbot** khi:

* Prompt không rõ → loop vô hạn
* Tool lỗi → toàn bộ pipeline fail
* Input user mơ hồ → agent không biết xử lý

Trong khi chatbot vẫn có thể “trả lời tạm”.

---

### 3. Observation

Observation đóng vai trò cực kỳ quan trọng:

* Là “feedback loop” cho LLM
* Quyết định bước tiếp theo
* Giảm sai lệch suy luận

Ví dụ:

```
Observation: Flight price = 2,000,000
→ Thought: Too expensive → try bus/train
```
